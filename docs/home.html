About this project

Record the state or behavior of live Java objects for playback during unit testing.

This tool was developed to help increase test coverage. The ThOR project currently has 98% unit test coverage as reported by EclEmma.

The latest release is version 1.4.0. Recent changes include:

  • Addition of the MockBeanRecorder
  • ClassWriter.setSuperClass() method to control the factory super class
  • MockBehaviorRecorder.setNice() method to create "nice" mock objects instead of the default "strict" mocks
  • Default support for BigDecimal and BigInteger object creation in state recorders
  • Objenesis object creation support in state recorders
  • BeanRecorder.stopDescent( Class ) method to prevent inspection from continuing descent on a class

Getting started

Two types of object recorders are provided, state and behavior recorders.

Download the zip file and add all jars to your classpath, then follow the links to see how to record objects and what to expect.

    Object state recorders:
  • BeanRecorder - requires Objenesis
  • MockBeanRecorder - requires Objenesis and EasyMock
  • XstreamRecorder - requires XStream and Objenesis
    Object behavior recorder:
  • MockBehaviorRecorder - requires EasyMock

See the Tutorials to learn how to use more advanced features.
